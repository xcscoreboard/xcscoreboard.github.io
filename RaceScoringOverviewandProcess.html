Race Scoring Overview and Process

Scoring/Timing


In order to accurately score and time a race in the xcscoreboard app, you need to ensure the following:


  1. Before the race, ensure all racers are registered accurately -- that each are assigned to the correct team, correct group, and that each affixes the correct assigned race bib to the front of their jersey. The registration task is done while xcscoreboard is in standalone mode.
      • Note that a race bib can be assigned to one and only one race group. To allow racers to optionally run in multiple groups (e.g., a runner may opt to run Varsity or JV), then either have two different race files for the event, assign those runners multiple bibs, each to a different race, or provide, a pre-event scratch/change meeting for coaches to re-assign registrations.
      • If using a timing system, a the Chip/Bib Mapping cross-reference table needs to be populated that maps each runner's assigned bib# to the associated globally unique RFID Chip-ID that is read by the timing system.

  1. During a race, finishers' bib#s should be entered in the Bibs/Timing page in the order of race completion. This task is done while xcscoreboard is in the orchestration mode (bib entry mode) or is done automatically by the timing system. This cross references the Racers table to identify the placement of runners for the race.

  2. During a race, finishers' times should be entered in the Bibs/Timing page in the order of race completion. This task is done while xcscoreboard is in the orchestration mode (timing mode) using the stopwatch timer or by the optional timing system.

If all of the three previous tasks are performed correctly, xcscoreboard provides accurate projected and final team and individual results for each race group in real-time. Cross Country Scoreboard is built to ensure these tasks are performed as easily and efficiently as possible.
